# 博客结构与技术方案（Apple 风格）

## 1. 目标与定位
- **公开展示 + 自我复盘**：既能记录事件，也能产出观点文章。
- **结构化时间轴**：以「年-月-周」为主轴，适合长期追踪与回顾。
- **轻量可视化**：用于统计与趋势概览，保持克制与高级感。
- **视觉风格**：参考 Apple 官网的极简、留白、细致层级与高质感排版。

---

## 2. 信息架构（IA）

### 2.1 内容类型
1. **周记（核心）**
   - 时间：按 Year → Month → Week 聚合
   - 内容：事件列表 + 复盘 + 学习/产出 + 下周计划

2. **观点/文章（可独立）**
   - 主题：技术、思考、复盘、生活等
   - 可关联到某一周记

3. **标签 & 搜索**
   - 标签统一前缀：`tech/`、`life/`、`review/`
   - 搜索覆盖：标题、摘要、正文、标签

---

## 3. 页面结构建议

### 3.1 首页（时间轴 + 年份导航）
- 左侧：年份快速导航（2024 / 2025 / 2026）
- 主区：周记卡片时间轴
- 右侧（可选）：年度统计卡片（总周记数、文章数、标签趋势）

### 3.2 周记页（结构化内容）
- 顶部：标题（例如：2025 年 03 月第 2 周）
- 概览：本周关键词、心情指数、主题一句话
- 事件清单：时间 + 事件描述 + 标签
- 产出/学习：本周读书、代码、项目总结
- 复盘：高光/低谷/改进点
- 下周计划：3 件以内重点目标
- 关联文章：本周产出文章列表

### 3.3 文章页（阅读体验优先）
- 统一标题区 + 摘要 + 标签
- 可选：目录导航（长文）
- 底部：关联周记 + 相关文章

### 3.4 标签页 / 搜索页
- 标签云 / 标签列表
- 搜索结果：高亮命中字段

---

## 4. 视觉风格（参考 Apple 官网）

### 4.1 视觉关键词
- 简洁、秩序、留白、克制
- 细线分隔、低对比阴影、浅层级卡片
- 高品质字体排版

### 4.2 色彩建议
- 背景：#0D0D0D / #111111（深色模式）
- 主色：#FFFFFF（文本）
- 点缀：#5A67D8 / #8AB4F8（细节高亮）
- 分割线：#1F1F1F

### 4.3 组件风格
- 卡片：低对比边框 + 圆角 + 轻阴影
- 按钮：极简实心 / 描边
- 图表：细线、低饱和度

---

## 5. 技术栈建议（推荐）

### ✅ 推荐方案：Next.js + MDX + Tailwind
- **Next.js**：适合 SSG/ISR，SEO 友好，部署简单（Vercel）
- **MDX**：适合写作与内容组织，支持嵌入组件
- **Tailwind**：快速搭建 Apple 风格布局与细节
- **搜索**：
  - 轻量方案：`next-mdx-remote + lunr.js`
  - 进阶方案：`Meilisearch / Algolia`
- **内容管理**：
  - 本地 Markdown / MDX
  - 或接入 Contentlayer/Content-Collections

## 6. 数据结构草案

### 6.1 周记数据结构（MDX Frontmatter 示例）
```yaml
title: "2025-03-W2"
date: "2025-03-10"
year: 2025
month: 3
week: 2
tags: ["review/weekly", "life/health", "tech/react"]
summary: "本周完成了项目里程碑，开始系统复盘锻炼计划。"
```

### 6.2 文章数据结构（MDX Frontmatter 示例）
```yaml
title: "如何设计年-月-周的个人知识系统"
date: "2025-03-12"
tags: ["tech/knowledge", "review/method"]
summary: "从时间结构与复盘习惯出发，搭建可持续的个人知识库。"
```
